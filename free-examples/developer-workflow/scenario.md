# Developer Workflow - Scénario

## Contexte

Un développeur utilise Free Context MCP pour gérer ses connaissances techniques pendant son travail quotidien.

## Flux de Travail

1. **Création d'un Contexte de Projet**
   - Créer un contexte "Next.js App Architecture"
   - Ajouter une description et des tags

2. **Ajout de Mémoires Techniques**
   - Note: "Server Components vs Client Components"
   - Snippet: "useHookPattern avec dependencies"
   - Référence: "Documentation Next.js 14 sur App Router"

3. **Recherche Intelligente**
   - Chercher "server state management"
   - Trouver automatiquement les mémoires connexes

4. **Détection de Relations**
   - Le système détecte automatiquement les connexions entre mémoires
   - Visualiser le graphe de connaissances

## Mémoires Créées

1. **Note**: "Server Components vs Client Components"
   - Type: note
   - Contenu: Explication des différences et cas d'usage

2. **Snippet**: "useHookPattern avec dependencies"
   - Type: snippet
   - Contenu: Code TypeScript pour un pattern React personnalisé

3. **Reference**: "Next.js 14 App Router Docs"
   - Type: reference
   - Contenu: Lien vers la documentation officielle

## Résultat Attendu

- Contexte créé avec ID unique
- 3 mémoires ajoutées
- Relations automatiques détectées
- Recherche FTS5 fonctionnelle
- Graphe de connaissances visualisable
